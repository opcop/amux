# Amux Agent 通知系统 + 电子宠物

> 创建日期: 2026-04-02
> 状态: 方案探讨阶段

---

## 1. 问题

用户跑多个 Agent 写代码时，经常最小化 Amux 去做别的事（查资料、看文档）。此时对 Agent 状态完全失明：

- 不知道哪个 Agent 完成了
- 不知道哪个 Agent 在等待输入
- 不知道哪个 Agent 报错了
- 要频繁切回 Amux 查看，打断工作流

Windows 系统通知中心不适合——各种应用通知混在一起，Agent 状态容易被淹没。

## 2. 方案：迷你悬浮窗 + 电子宠物

Amux 最小化后，在屏幕角落显示一个常驻悬浮窗，以电子宠物为载体展示 Agent 状态。

### 2.1 基础形态

```
平时（宠物待机）：

    ╭──╮
    │🐱│  ← 宠物有 idle 动画
    ╰──╯

Agent 状态变化时（气泡通知）：

    ╭──╮
    │🐱│  "Claude 写完了喵~"
    ╰┬─╯
     │   ✓ Claude Code    2m30s
     │   ⟳ Codex         1m05s
     │   ● OpenCode  等待输入  [回复]
```

### 2.2 宠物行为与 Agent 状态联动

| Agent 状态 | 宠物行为 |
|-----------|---------|
| 所有 Agent 空闲 | 待机动画（睡觉、打盹、玩耍） |
| 有 Agent 在工作 | 忙碌动画（敲键盘、翻书、写代码） |
| Agent 完成任务 | 庆祝动作 + 气泡通知 |
| Agent 等待输入 | 求关注动作（招手、跳跃） |
| Agent 报错 | 困惑/求助表情 |

### 2.3 交互功能

- **点击宠物**：展开/收起 Agent 状态列表
- **点击某个 Agent 行**：直接跳回 Amux 对应的 pane
- **回复输入框**：Agent 等待输入时，直接在悬浮窗里输入回复发送回去，不用切回 Amux
- **Amux 恢复窗口时**：悬浮窗自动隐藏
- **Amux 最小化时**：悬浮窗自动显示

### 2.4 极简模式（可选）

不喜欢宠物的用户可以切换为纯文字状态条：

```
┌─ AMUX ──────────────────┐
│ ✓ Claude Code    2m30s  │
│ ⟳ Codex         1m05s  │
│ ● OpenCode  等待输入     │
└─────────────────────────┘
```

## 3. 商业模式：宠物商店

**核心原则：工具完全免费，宠物纯装饰付费。** 免费用户和付费用户的 Agent 监控能力完全一样，只是悬浮窗的宠物不同。

### 3.1 定价

| 层级 | 价格 | 内容 |
|------|------|------|
| 默认宠物 | 免费 | 1 个基础像素猫（随 Amux 自带） |
| 宠物商店 | $1-3/只 | 不同品种和风格 |
| 季节限定 | $2-5/只 | 节日限定款、联名款 |
| 宠物包 | $5-10 | 一套主题系列 |

### 3.2 宠物类型示例

| 系列 | 示例 | 风格 |
|------|------|------|
| 经典动物 | 像素猫、像素狗、仓鼠、企鹅 | 像素风 |
| 编程语言 | Rust 螃蟹🦀、Python 蛇🐍、Go 地鼠 | 拟人化吉祥物 |
| 赛博朋克 | 机械猫、霓虹狐、电子蝴蝶 | 科幻风 |
| 国风 | 招财猫、锦鲤、熊猫 | 中国风 |
| 节日限定 | 圣诞驯鹿、万圣南瓜猫、春节舞狮 | 季节限定 |

### 3.3 商业逻辑

单靠宠物收费体量小，但它的真正价值：

1. **用户粘性** — 养了宠物就不想换工具（沉没情感）
2. **社交传播** — "你的宠物好可爱，什么工具？" → 自然获客
3. **付费习惯** — 有了小额付费基础，后续 Pro 订阅更容易推
4. **品牌辨识** — 宠物成为 Amux 的品牌符号

可与 Pro/Team 订阅（Compare/Pipeline 等高级功能）并行：工具高级功能走订阅，宠物走小额付费，两条腿走路。

## 4. 竞品参考

### Vibe Island (macOS, $19.99)

- 利用 macOS 刘海屏（Dynamic Island）做 Agent 状态面板
- 支持 6 个 Agent 同时监控
- 可以在通知面板里直接回复 Agent 提问
- 权限审批 GUI 化（Allow/Deny 按钮）
- 通过 Unix socket + hooks 抓取 Agent 数据

**Amux 的差异化**：
- Amux 是终端本身，不是外挂工具，集成更深
- 不需要 socket 桥接，Agent 输出直接在 pane 里
- 宠物 + 情感化设计是 Vibe Island 没有的维度
- 跨平台（Windows 为主，未来 macOS/Linux）

## 5. 技术实现思路

### 5.1 悬浮窗

- GPUI 支持创建多个窗口，悬浮窗作为第二个 GPUI 窗口
- 窗口属性：always-on-top、无边框、透明背景、鼠标穿透（宠物区域外）
- 监听 Amux 主窗口的最小化/恢复事件来自动显隐

### 5.2 宠物渲染

- Sprite sheet 动画（PNG 序列帧），GPUI canvas 绘制
- 每个宠物定义一组动画状态（idle、busy、celebrate、confused、sleep）
- 宠物资源打包为 `.amux-pet` 文件（zip 包含 sprite sheet + manifest.json）

### 5.3 状态数据

- 复用现有 `AgentStatus` 检测（poll_activity）
- 主窗口通过共享状态（Arc<Mutex>）传递给悬浮窗
- 或通过 GPUI 的跨窗口事件机制

### 5.4 宠物商店

- 宠物资源托管在 CDN
- Amux 内置商店浏览页面（列表 + 预览动画）
- 购买通过 Stripe/Paddle 等支付平台
- 购买后下载 `.amux-pet` 到 `~/.amux/pets/`

## 6. 风险与对策

| 风险 | 对策 |
|------|------|
| 开发者觉得"不够严肃" | 宠物默认关闭，用户主动开启；提供极简模式 |
| 宠物开发成本高（美术资源） | 先出 1 个免费像素猫验证需求，后续众包或 AI 生成 |
| 付费转化率低 | 宠物只是获客手段，核心收入靠 Pro/Team 订阅 |
| 悬浮窗被系统拦截 | 提供 fallback：任务栏图标 + tooltip 状态摘要 |
