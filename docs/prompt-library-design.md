# Prompt Library — 提示词知识库设计

## 核心命题

Vibe Coding 开发者最有价值的资产不是代码，而是**和 AI 沟通的能力**——即提示词。但这些提示词散落在无数次对话里，用完即弃。

```
Session 1: "帮我重构这个函数，用策略模式替代 if-else，保持接口不变" → 效果很好
Session 2: "帮我重构..." → 忘了上次怎么写的，重新摸索
Session 3: 完全忘了有过这个好用的提示词
```

Amux 坐在用户和 AI 之间，能看到所有输入输出。不需要用户主动做任何事情，就能默默积累提示词知识库。

## Amux 的天然优势

Amux 已经具备实现这个功能的全部基础设施：

| 基础能力 | 用途 |
|---------|------|
| Agent 检测（AgentKind/AgentStatus） | 判断当前 pane 是否在和 AI 对话 |
| PTY 输入监控 | 捕获用户输入的提示词文本 |
| PTY 输出监控 | 评估 AI 回复质量（成功/失败信号） |
| Agent Bridge | 跨 pane 共享提示词推荐 |
| Sidebar UI | 可视化浏览提示词库 |
| 终端命令拦截 | `amux prompt` 命令管理提示词 |

**没有任何其他工具在做这件事**——Claude Code 不记录提示词历史，Cursor/VS Code 没有提示词管理，Prowl/tmux 更不可能有。只有终端多路复用器才能同时看到用户输入和 Agent 输出，从而评估提示词效果。

## 功能设计

### 1. 自动捕获

当 Amux 检测到 pane 里运行的是 AI Agent（Claude/Codex/Aider 等），且 Agent 状态为 `Waiting`（等待输入）时，用户的下一段输入就是提示词。

**捕获规则：**
- Agent 状态为 Waiting → 用户输入 → Enter 提交 = 一条完整提示词
- 过滤：单字符输入（y/n 确认）、纯路径、纯 shell 命令
- 过滤：包含敏感关键词的输入（可配置排除规则）
- 多行输入合并为一条提示词

**存储：**
```
~/.amux/prompts/
  library.json          # 提示词库
  stats.json            # 使用统计与评分
```

### 2. 自动分类

根据提示词内容自动打标签：

| 关键词模式 | 标签 |
|-----------|------|
| 重构、refactor、优化结构 | #refactor |
| 测试、test、写用例 | #testing |
| 修复、fix、bug、报错 | #debugging |
| review、审查、检查 | #review |
| 设计、架构、方案 | #architecture |
| 文档、注释、说明 | #documentation |
| 性能、优化、提速 | #performance |
| 安全、漏洞、权限 | #security |

基于关键词匹配实现，纯本地，不需要 AI 参与。

### 3. 质量评估与打分

自动评估提示词效果，从多个维度打分（每项 1-5 分）：

| 维度 | 衡量方式 | 信号来源 |
|------|---------|---------|
| **复用频率** | 语义相似的提示词出现次数 | 历史比对 |
| **一次成功率** | Agent 回复后是否需要反复修改 | 后续输入分析 |
| **长度效率** | 越短效果越好 = 高分 | 字符统计 |
| **用户满意度** | 用户反馈信号 | 输出文本分析 |

**正面信号（提升评分）：**
- Agent 回复后用户说"好的"/"完美"/"正确"/"可以"
- Agent 回复后用户直接执行 `git commit`（说明结果满意）
- Agent 一次完成任务，没有后续修改请求
- 提示词被再次使用（复用 = 认可）

**负面信号（降低评分）：**
- Agent 回复后用户说"不对"/"重新"/"错了"/"不是这个意思"
- 同一任务连续发送 3+ 条提示词（说明前面的不好用）
- Agent 回复后用户使用 `git checkout`/`git reset`（说明结果不满意）

**综合评分公式：**
```
score = (reuse_count × 2 + success_rate × 3 + efficiency × 1 + satisfaction × 4) / 10
```

满意度权重最高，因为用户的直接反馈最可靠。

### 4. 搜索与复用

#### 命令行方式

```bash
# 保存当前提示词（手动标星）
amux prompt save

# 列出所有保存的提示词（按评分排序）
amux prompt list

# 按关键词搜索
amux prompt search "重构"

# 按标签筛选
amux prompt list --tag refactor

# 查看统计
amux prompt stats
```

### 5. 提示词模板

高频提示词支持变量化，变成可复用的模板：

```
模板："帮我为 {file} 写单元测试，覆盖边界情况和错误处理"
使用时：自动替换 {file} 为当前文件名

模板："review {file}，重点检查 {focus}"
使用时：{file} = 当前文件，{focus} = 用户输入
```

**模板变量：**

| 变量 | 自动填充内容 |
|------|-------------|
| `{file}` | 当前活跃文件 / 最近修改的文件 |
| `{dir}` | 当前工作目录 |
| `{branch}` | 当前 git 分支 |
| `{error}` | 最近的错误输出 |
| `{selection}` | 终端选中的文本 |
| `{pane}` | 当前 pane ID |

## 隐私保护

提示词可能包含敏感信息。隐私策略：

| 措施 | 说明 |
|------|------|
| **纯本地存储** | 所有数据存在 `~/.amux/prompts/`，永不上传 |
| **手动删除** | `amux prompt delete <id>` 删除任意提示词 |
| **排除规则** | 包含配置关键词的提示词不记录（如 API key、password） |
| **自动过期** | 超过 N 天未使用的低分提示词自动清理 |
| **加密选项** | 可选对提示词库加密存储 |

配置：

```toml
# ~/.amux/config.toml

[prompt_library]
enabled = true
# 自动捕获（关闭则只有手动 save 的会记录）
auto_capture = true
# 排除包含这些关键词的提示词
exclude_keywords = ["password", "secret", "api_key", "token", "credential"]
# 低分提示词自动清理天数（0 = 不清理）
auto_cleanup_days = 90
# 最大存储条数
max_prompts = 1000
# 自动推荐（输入时弹出历史匹配）
auto_suggest = true
# 最低推荐评分
min_suggest_score = 3.0
```

## 语义相似度实现

判断两条提示词是否"相似"（用于去重和复用计数），轻量本地方案：

### 方案：关键词 + 编辑距离混合

```rust
fn similarity(a: &str, b: &str) -> f32 {
    let keyword_sim = keyword_overlap(a, b);    // Jaccard 系数
    let edit_sim = normalized_edit_distance(a, b); // 归一化编辑距离
    keyword_sim * 0.6 + edit_sim * 0.4
}
```

- **关键词重叠**（Jaccard 系数）：提取中文/英文关键词，计算交集/并集
- **归一化编辑距离**：字符级别相似度
- 阈值 > 0.7 = 判定为相似提示词

**不需要 AI 参与**，纯本地计算，毫秒级响应。

## 与 Amux 其他能力的结合

| 现有能力 | 与提示词库的协同 |
|---------|----------------|
| **Agent Bridge** | 提示词库跨 pane 共享——pane-1 用过的好提示词推荐给 pane-2 的 agent |
| **Agent 状态检测** | Waiting 状态自动触发提示词推荐弹窗 |
| **代码卫生提醒** | 提醒"上次用 /simplify 效果很好"并附带当时的完整提示词 |
| **Auto-Feed** | 调试循环中自动记录有效的调试提示词 |
| **Session 录制** | 录制时标记高效提示词，回放时高亮显示 |
| **Orchestrator 工作流** | 工作流步骤引用提示词库中的模板 |

## 产品演进路径

### 短期：最小可行版本

- `amux prompt save` 手动保存当前提示词
- `amux prompt list` / `amux prompt search` 浏览和搜索
- 本地 JSON 存储
- 工作量：2-3 天

### 中期：自动化版本

- 自动捕获（Agent Waiting → 用户输入 → 自动记录）
- 自动分类（关键词标签）
- 质量评分（复用/成功率/满意度）
- Sidebar Prompts 视图
- 输入时自动推荐弹窗
- 工作量：5-7 天

### 长期：智能版本

- 提示词模板（带变量自动填充）
- 团队提示词库共享
- 提示词 A/B 测试（同一任务两种写法对比效果）
- 提示词进化（基于使用反馈自动优化措辞）
- 提示词市场（社区分享高分提示词）
- 工作量：2-4 周

## 产品定位升级

```
当前：Amux = 多 Agent 协作操作系统
加入提示词库后：Amux = 多 Agent 协作操作系统 + 提示词知识引擎

开发者使用 Amux 越久：
  → 积累的提示词越多
  → 提示词质量越高（评分淘汰低效的）
  → 开发效率越高
  → 越离不开 Amux（正向飞轮）
```

**护城河**：用户的提示词库是个人知识资产，积累越多越难迁移。这是 Amux 最强的用户粘性来源。
